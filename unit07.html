Constructing Normalised Tables and Database Build

We examined an un-normalized table and systematically transforming it into 1NF, 2NF, and 3NF, illustrating each step of the process. The obtained data were utilized to construct a relational database using the application environment of our choice.

Unit 7 covers:

  • Examining data attributes, associations, operations and relationships.
  • Normalising a set of data and Identify constraints associated with the application of the data attributes.
  • Constructing a relational database model based on normalised data.
  • Testing the database to check for any errors or anomalies.

Learning Outcomes:

  • To understand the concept of transforming a flat file database model into a normalised model.
  • To build a relational database model and test its useability.
  • To identify functional dependencies of data items in the dataset.
  • To evaluate constraints and limitations associated with the use and application of a dataset.
  • To specify data attributes relevant and critical to using a given dataset.
